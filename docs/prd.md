# 租房对比地图工具 — PRD

> 版本：v0.2 MVP · 个人工具，单用户

---

## 一、产品背景与目标

租房决策需要同时权衡多个维度：租金、通勤距离、周边配套、房屋本身条件。房源信息分散在中介微信、实地看房记录中，缺乏统一的对比视图。

**目标**：建立一个个人租房决策工具，以地图 + 列表双视角呈现 10 套以内的房源，核心价值是接入高德地图数据，让位置、通勤、周边配套可量化对比。

---

## 二、用户与使用场景

| 项目 | 说明 |
|------|------|
| 用户 | 仅限本人，单用户工具 |
| 使用设备 | iPhone（外出看房）/ MacBook Pro（室内决策分析） |
| 房源规模 | 上限约 10 套 |
| 使用周期 | 一次性使用（租房周期内） |

## 三、核心流程

### 流程一：添加房源（两阶段录入）

```
阶段一（看房前）
中介微信发视频
  → 手机保存视频到相册
  → 打开 web app，新建房源
  → 填写：名称、大概地址（或地图选点）
  → 上传视频到 TOS
  → 保存，状态标记为「待看」

阶段二（看房后）
实地看完
  → 打开对应房源，进入编辑
  → 补充：租金、面积、楼层、朝向、装修情况、备注
  → 状态改为「已看」或「排除」
```

### 流程二：通勤分析

```
设置目标地点（如公司地址）
  → 系统调用高德公共交通路径规划 API
  → 地图上每个房源 pin 显示通勤时间
  → 列表中通勤时间列同步更新
```

### 流程三：查看房源详情

```
点击地图 pin 或列表某一行
  → 弹出 / 跳转详情面板
  → 展示：视频、基本信息、通勤时间、周边 POI
  → 可直接编辑或修改状态
```

## 四、页面设计

### 4.1 主界面布局

**PC 端（宽屏）：左右分栏**

```
┌──────────────────────────────────────────────────────┐
│  顶部栏：目标地点设置 / 筛选条件 / 添加房源按钮         │
├─────────────────────┬────────────────────────────────┤
│                     │                                │
│    高德地图          │    房源列表                    │
│    · 房源 pin        │    · 每行一套房源               │
│    · 通勤时间标注    │    · 列：名称/租金/面积/通勤    │
│    · 到达圈图层      │    · 点击行 → 地图联动定位     │
│                     │    · 支持排序                  │
│                     │                                │
└─────────────────────┴────────────────────────────────┘
```

联动规则：
- 点击地图 pin → 右侧列表对应行高亮，滚动到可见区域
- 点击列表某行 → 左侧地图飞到对应房源位置，pin 高亮
- 两侧共用同一个「当前选中房源」状态

**手机端（窄屏）：Tab 切换**

```
┌──────────────────────┐
│  顶部栏（同 PC）      │
├──────────────────────┤
│  [ 列表 ] [ 地图 ]    │  ← Tab，默认列表
├──────────────────────┤
│                      │
│  列表视图 or 地图视图  │
│                      │
└──────────────────────┘
```

### 4.2 房源详情面板

PC 端：点击 pin 或列表行，右侧列表区域替换为详情面板（可返回列表）
手机端：全屏底部抽屉或跳转新页面

详情面板内容：
- 视频播放器（TOS 直链）
- 基本信息：地址、租金、面积、楼层、朝向、装修、状态
- 通勤信息：到目标地点的公共交通方案摘要（总耗时，含步行接驳，高德数据）
- 周边 POI：地铁站、超市、医院（可按类别切换，高德数据）
- 备注
- 编辑 / 删除按钮

### 4.3 添加 / 编辑房源表单

字段如下：

| 字段 | 类型 | 阶段 | 说明 |
|------|------|------|------|
| 房源类型 | 下拉选择 | 一 | 小区房 / 城中村 / 独栋公寓（城中村）/ 独栋公寓（非城中村）|
| 房型 | 下拉选择 | 一 | 单间 / 大单间 / 一房一厅 / 二房一厅 / Loft |
| 名称 | 文本 | 一 | 如「朝阳某小区 2室1厅」 |
| 地址 | 文本 + 地图选点 | 一 | 输入后自动地理编码定位 |
| 视频 / 图片 | 文件上传 | 一 | 上传至 TOS，支持多个 |
| 租金 | 数字（元/月） | 二 | |
| 面积 | 数字（㎡） | 二 | |
| 楼层 | 文本 | 二 | 如「6/18」 |
| 朝向 | 下拉选择 | 二 | 南 / 北 / 东 / 西 / 东南 / 其他 |
| 装修 | 下拉选择 | 二 | 精装 / 简装 |
| 有无燃气 | 是 / 否 | 二 | |
| 燃气费 | 数字（元/方） | 二 | 有燃气时填写 |
| 电费 | 数字（元/度） | 二 | |
| 水费 | 数字（元/吨） | 二 | |
| 管理费 | 数字（元/月） | 二 | 无则填 0 |
| 网络 | 下拉选择 + 备注 | 二 | 包网（注明费用）/ 可自接 / 不可接 |
| 冰箱 | 文本 | 二 | 如「无」「双门小冰箱」「大双开门」 |
| 含阳台 | 是 / 否 | 二 | |
| 有无车位 | 下拉选择 | 二 | 有（注明费用）/ 无 |
| 备注 | 多行文本 | 二 | 自由填写 |

系统自动计算字段（无需手动填写）：
- 单价（元/㎡）= 租金 ÷ 面积
- 通勤时间（分钟）= 高德公共交通方案总耗时（默认地铁优先，含公交和步行接驳），目标地点变更时批量刷新
- 通勤时间仅展示单一总耗时，不拆分展示地铁 / 公交 / 步行明细

## 五、功能需求

### 5.1 房源管理

- 新建房源，支持两阶段分次填写
- 编辑任意字段
- 删除房源（二次确认）

### 5.2 地图功能

- 所有房源在地图上显示 pin，pin 颜色按状态区分
- 点击 pin 展示简要信息气泡（名称、租金、通勤时间）
- 目标地点分两种模式：

  **固定目标**（持久化）
  - 保存一个主目标地点（如公司），存入 `targets.json`（当前激活目标存入 `settings.json`）
  - 通勤时间计算后缓存在每个房源数据里
  - 列表和地图 pin 常驻显示该通勤时间
  - 固定目标变更时，批量重新请求高德 API 刷新缓存

  **临时查询**（一次性）
  - 顶部搜索框随时输入任意地址（如机场、某商圈）
  - 实时调用高德 API 计算所有房源到该地点的通勤时间
  - 结果覆盖显示在列表和 pin 上，但不写入数据
  - 临时查询激活时，顶部显示提示条：「当前显示到：XX 的通勤时间」并附「清除」按钮
  - 清除查询后恢复显示固定目标的通勤时间

- 到达圈（P1）：以当前激活的目标地点为中心，绘制 X 分钟公共交通可达范围

### 5.3 列表功能

- 展示所有房源，每行显示核心字段：名称、租金、面积、单价、通勤时间、朝向、装修、有无燃气、含阳台
- 点击列头排序（租金、面积、单价、通勤时间）
- 列表行点击进入详情
- 顶部筛选条件（P0）：状态（待看 / 已看 / 排除）、租金区间、面积区间、房型、房源类型

### 5.4 周边 POI 查询（P1）

在详情面板内，以房源为中心查询周边：
- 地铁站（显示距离）
- 超市 / 便利店
- 医院
- 数据来源：高德 PlaceSearch.searchNearBy

### 5.5 媒体管理

- 上传视频 / 图片至 TOS，支持一个房源多个文件
- 详情面板内播放视频、浏览图片
- 删除媒体文件

### 5.6 到达圈（P1）

- 以当前激活的目标地点为中心，在地图上绘制 X 分钟公共交通可达范围
- X 分钟可由用户输入（如 30 分钟、45 分钟）
- 数据来源：高德 ArrivalRange API

### 5.7 多目标地点（P1）

- 支持保存多个目标地点（如公司 + 常去商圈）
- 列表可切换当前显示哪个目标地点的通勤时间
- 临时查询功能保留，独立于多目标地点之外

### 5.8 AI 房源评估（P1）

- 单套评估：对某一房源生成优劣分析，结合所有字段 + 高德通勤 / POI 数据，输出自然语言总结
- 多套对比：选中 2 套或以上房源，由 LLM 横向对比各维度差异，给出综合建议
- LLM 调用走 Hono `/api/ai/evaluate` 路由，支持 Claude 或豆包，模型可配置
- 分析结果展示在详情面板或独立对比页，不持久化存储

## 六、数据模型

```ts
interface Property {
  id: string
  name: string
  address: string
  lat: number
  lng: number

  propertyType?: 'residential' | 'urban-village' | 'standalone-urban' | 'standalone'  // 小区房 / 城中村 / 独栋公寓(城中村) / 独栋公寓(非城中村)
  roomType?: 'studio' | 'large-studio' | '1b1l' | '2b1l' | 'loft'  // 房型
  area?: number            // 面积，㎡
  floor?: string           // 如 "6/18"
  orientation?: string     // 朝向
  decoration?: 'luxury' | 'simple'
  hasGas?: boolean         // 有无燃气
  gasFee?: number          // 燃气费，元/方
  electricityFee?: number  // 电费，元/度
  waterFee?: number        // 水费，元/吨
  managementFee?: number   // 管理费，元/月，无则为 0
  network?: 'included' | 'self' | 'unavailable'
  networkNote?: string     // 包网费用或自接说明
  fridge?: string          // 冰箱描述
  hasBalcony?: boolean     // 含阳台
  parking?: 'available' | 'unavailable'
  parkingNote?: string     // 车位费说明
  note?: string

  // 系统计算
  pricePerSqm?: number     // 单价，自动计算

  // 媒体
  mediaUrls: string[]      // TOS 直链

  // 高德数据缓存
  commuteCache?: {
    targetId: string
    minutes: number
    mode: 'transit'       // 固定为公共交通方案（默认地铁优先）
    updatedAt: string
  }[]
  // 缓存仅保存单一总耗时（minutes），不额外持久化地铁/公交/步行分项

  createdAt: string
  updatedAt: string
}

interface Target {
  id: string
  name: string            // 如「公司」
  address: string
  lat: number
  lng: number
}

// 临时查询结果仅存于前端状态，不持久化
interface TempQuery {
  address: string
  lat: number
  lng: number
  results: { houseId: string; minutes: number }[]
}
```

## 八、MVP 范围界定

**MVP 包含（P0）：**
- 房源增删改、两阶段录入
- 地图展示 + 列表展示 + 联动
- 目标地点设置 + 通勤时间计算
- 视频 / 图片上传与播放
- PC 左右分栏 + 手机 Tab 切换

**MVP 之后（P1）：**
- 到达圈（ArrivalRange）
- 周边 POI 查询
- 多目标地点支持
- AI 房源评估：调用 LLM（Claude / 豆包）对单个房源生成优劣分析，或对多套房源横向对比，结合所有结构化字段 + 高德数据输出自然语言总结，辅助最终决策
